1、函数中的arguments方法用于不定参数场合的使用，代表实参的集合，以类数组的形式展现​
2、当实参比形参的个数多时，按顺序接收，多余的则不输出；当实参比形参的个数少时，没有实参的形参的值为undefined​
3、当遇到同名标识符时，预解析阶段会把标识符给函数​
        test();
        var test = 8;
        console.log(test);
        function test() { // 优先 提升
            console.log('生活可真不容易');
        }
        //输出结果为 生活可真不容易  8
        //由此可看出同名标识符的情况下 会给函数而不是给变量

25、原型对象​
(1)所有引用类型都有一个 __proto__(隐式原型)属性,属性值是一个普通对象
(2)所有函数都有一个prototype(原型)属性,属性值是一个普通的对象
(3)所有引用类型的 __proto__属性都是指向它 构造函数的 prototype
(4)构造函数需要原型对象的原因：让多个实例之间通用方法使用一个引用，减少空间的占用
26、继承​

​ 原型链继承

​ 核心：原型链对象编程父类的实例，子类就可以调用父类的方法和属性

​ 缺点：引用类型值的原型属性会所有实例共享 ； 不能向父类传递参数

    var son1 = new Son('金志成',20);
    var son2 = new Son('小婉婷',21);

    console.log(son2.friends); //输出的结果包含小婉婷和Number

​ 组合模式继承

​ 核心：子类的构造函数内部调用父类的构造函数，并且传入this指针

​ 优点：解决了原型链继承的问题

​ 缺点：在任何情况下都会调用2次父类的构造函数

    Son.prototype=new Father()
    Son.prototype.constructor=Son
    function Son(){
        Father.call(this,svg) //此时父类的this为Son的this
    }

​ 寄生组合继承

​ 核心：需要创建一个临时的构造函数，并将父类的对象作为构造函数的原型，返回一个新对象父类的属性，对于子类来说是共享的

    Son.prototype=Object.create(Father.prototype) //避免执行构造函数  减少一次父类构造函数的执行
    Son.prototype.constructor=Son
    function Son(){
        Father.call(this,svg) //此时父类的this为Son的this
    }

33、深浅拷贝​
深拷贝：​

​ 相当于创建一个一模一样的对象，新的对象和原来的对象之间不共享内存，修改对象不会改变原对象

​ 方法1：使用JSON.stringify和 JSON.parse 序列化和反序列化来完成深拷贝

​ 原理：用 JOSN.stringify 方法 将 一个JavaScript的值(对象或者数组)转换为一个JSON字符串，再用 JSON.parse 把字符串解析成对象,新的对象就产生了,而且会开辟新的栈(内存空间),实现深拷贝

​ 方法2：递归实现

    var obj = {name:'民圣',age:16,money:[100,[1000,10000]],say:function(){},sc:                           {sex:125,names:"哈哈"}}               
    var deep = deepCopy(obj);
    deep.money[1] = ['1k','10k'];
    function deepCopy(obj){
        // 判断是数组还是对象
        var objClone=Array.isArray(obj)?[]:{};
        // 首先判断obj是否是对象且存在
        if(obj&&typeof obj==='object'){
            for(var key in obj){
                // 判断对象属性是否是对象自身的属性
                if(obj.hasOwnProperty(key)){
                // 如果属性值为object 则需要递归函数 直至是基本类型
                // 此举是为了避免赋值属性与obj属性的的内存地址相同
                    if(obj[key]&& typeof obj[key]==="object"){
                        objClone[key]=deepCopy(obj[key]);
                    }else{
                        objClone[key]=obj[key]
                    }
                }
            }
        }
        return objClone
    }

36、箭头函数​

​ 格式: ()=>{}

特征：     1. 没有绑定 this 
          2. 不能使用 arguments,对象在函数体内不存在. 解决: 可以用rest参数代替 ,arguments
                function fn(...args){}  fn(1,2,3,4)
          3. 不能把箭头函数当成 构造函数来使用, 箭头函数 不能使用 new 命令,否则会报错
          4. 不可以通过 call apply bind 修改 this 值

​ 函数体中 只有 一句代码,且代码执行结果就是返回会值,就可以省略 大括号 和 return

    let show = (a,b) => a + b;
    console.log(show(10,20));

​ 当形参只有一个时，就可以省略圆括号和return和大括号

    const fn = c => c;
    console.log(fn(10));

​ 当想要返回一个对象时，不能单用大括号 会被定义成代码块

    const obj = id => ({id:id, name:"test"});
等同于  
    function obj(id){
            return {id:id,name:"test"}
        }

​ 箭头函数中的this指向是 作用域中的this指向谁，箭头函数中的this就指向谁

39、Iterator接口问题​

​ iterator作用：为各种数据结构,提供一个统一的简便访问接口；使 数据结构的成员能够按照某种次序排序； 可以供 for of 使用

46、async​

​ 异步函数对象

​ 例子

    async function testAsync(){
    console.log('1');
    let p = new Promise((resovle,reject)=> setTimeout(resovle,1000,"promise对象"));
    console.log(3);
    // await 会造成代码的阻塞, 只有等待promise返回结果 , 等待完成才会执行下一步
    console.log('4',await p);// 1 秒之后 打印 promise对象

    }

​ await用于接收promise返回结果，不能单独使用，必须放在async函数中，否则会报错

​ 当await接收的是失败状态返回的结果时，可以通过try catch来捕获异常

         async function getNum(){
            let p = new Promise((resovle,reject)=>{
                // resovle("这是成功调用的函数");
                reject("这是失败调用的函数")
            })
            //当p接收的是成功返回的数时，会执行try中的代码
            //当p接收的是失败返回的数时，会执行catch中的代码
             try{

                // 右侧是其他类型的数据(非promise)
                let num = await 20;
                console.log(num);
            }catch(err){
                console.log(err);
            }
        }
